# Work Scholar Backend

This is the backend project for the Work Scholar application, built using the Laravel framework and PostgreSQL database.

## Project Structure

The project follows the standard Laravel directory structure, which includes:

- **app/**: Contains the core application logic, including models, controllers, middleware, and requests.
- **bootstrap/**: Contains files for bootstrapping the application.
- **config/**: Contains configuration files for the application.
- **database/**: Contains migration, factory, and seeder files for database management.
- **public/**: Contains the entry point for the application and publicly accessible assets.
- **resources/**: Contains views and language files for the application.
- **routes/**: Contains route definitions for the application.
- **storage/**: Contains files generated by the application, including logs and cached data.
- **tests/**: Contains unit and feature tests for the application.

## Installation

1. Clone the repository:
   ```
   git clone <repository-url>
   ```

2. Navigate to the project directory:
   ```
   cd work-scholar-backend
   ```

3. Install dependencies:
   ```
   composer install
   ```

4. Set up the environment file:
   ```
   cp .env.example .env
   ```

5. Update the `.env` file with your database credentials and other environment settings.

6. Generate the application key:
   ```
   php artisan key:generate
   ```

7. Run migrations to set up the database:
   ```
   php artisan migrate
   ```

8. (Optional) Seed the database with initial data:
   ```
   php artisan db:seed
   ```

## Usage

To start the development server, run:
```
php artisan serve
```

You can access the application at `http://localhost:8000`.
